## Codebase Patterns
- Use `lib/data/battleFormats.ts` as the single source of truth for format ids + metadata (`cup`, `cp`, labels) so UI, API, loaders, and sync logic stay aligned.
- For dropdowns, use `components/atoms/Select/Select.tsx` with `components/atoms/Select/Option.tsx` so floating labels and chevron indicators stay consistent in light/dark themes.
- Keep battle-format selection state in `components/organisms/TeamManager/TeamManager.tsx` and pass it down to `TeamConfigPanel`; render options via `getBattleFormats()` to avoid hardcoded UI/API drift.
- For battle-format API requests, send `formatId` from `TeamManager`, default missing values to `DEFAULT_BATTLE_FORMAT_ID` in routes, and validate with `isBattleFormatId` before calling `generateTeam`.
- In `lib/data/rankings.ts`, resolve CSV filenames from format metadata (`cp{cp}_{cup}_{category}_rankings.csv`) and keep lazy-load caches isolated per format id so one format failure never pollutes another.
- In `lib/genetic/algorithm.ts`, always pass `formatId` into `getTopRankedPokemonNames` and build candidate pools through `getRankedPokemonForFormat` so league/cup eligibility follows the selected format.
- In `lib/data/simulations.ts`, parse simulation files by format metadata (`cp` + `cup`), call `ensureSimulationDataAvailable(formatId)` before generation, and thread `formatId` through simulation helper calls to avoid non-Great fallback.
- For ranking sync, resolve PvPoke JSON paths through `createPvpokeAdapter().readRankingJson(category, cp, cup)` and emit deterministic filenames as `cp{cp}_{cup}_{category}_rankings.csv` for every supported battle format.
